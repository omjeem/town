Town Layout Playgroundexpansion algorithm preview · same seed = same town

Different users get different towns. Pick a seed, slide the stage to see how the town grows as plots come online.

What a "plot" is: an organic clearing carved out of an endless forest. There is no visible parcel border — the irregular shape of the clearing is the only cue that a plot owns this area. The building sits inside the clearing; bushes, flowers, and grass tufts fill the space between the building and the tree line.

How expansion works: The world is a dense forest by default. Plots come online in priority order: home → office → library → store → gym → cafe → studio → stage → practice → workshop → station. HOME always lands in the center; remaining plots get assigned by a deterministic shuffle, seeded by hash(seed, plotKey) — same user, same town. As each plot activates, an irregular clearing is carved out of the forest around it (radius noise per angle bucket, derived from the plot key), and a winding 2-tile-wide dirt road connects HOME to the new building via a seeded bezier. Decor inside each clearing is scattered by hash(seed, plotKey, slot) so two users with the same building still get visibly different yards.
